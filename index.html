JavaScript Promises

Promise is an object representing the current state of an asynchronous operation.

It serves as a wrapper for a value that is not known when the promise is created.

This approach enables handling the results of asynchronous operations as if they were synchronous: instead of directly returning the result of the asynchronous operation, a promise for the future result is provided.

1. States of a Promise

A promise can assume one of three states:

  • Pending: The initial state upon creating a promise.

  • Fulfilled: Indicates that the operation was completed successfully, yielding some result.

  • Rejected: Signifies that the operation was unsuccessful and was rejected with an error.
